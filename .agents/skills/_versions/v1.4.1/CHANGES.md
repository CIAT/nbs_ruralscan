# Ruleset v1.4.1 — 2026-07-20 (patch)

Clarification to catalogue #15 (no new rule/check). Defines the crop-vs-species split:

- **`crop_specific`** = a CULTIVATED cash/food crop (coffee, cocoa, mulberry, banana, citrus,
  groundnut, cereals, cultivated food trees e.g. breadfruit / Tahitian-chestnut / tropical-almond).
- **`species_specific`** = a tree grown for timber/service/fodder, OR a wild-harvested tree even
  if it fruits (baobab/Adansonia, sandalwood/Santalum, Faidherbia, merbau).

The product + cultivation decide, NOT tree-vs-not. Trigger: 2026-07 dashboard review — coffee
sat in the Species lane (tagged with the Coffea binomial) while mulberry was `crop_specific`, so
the Crop lane under-counted. 30 agroforestry rows reclassified species→crop under this rule
(Adansonia/baobab + Santalum/sandalwood kept species per Pete).
