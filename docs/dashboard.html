⚐ QA / QC — VERIFY-FLAG review Every EV row flagged by the relationship-verify pass — work through these before any threshold is trusted

MISMATCH — a value contradicted the quote (auto-corrected) UNSUPPORTED — a value the quote never contained (stripped) flagged = confidence-low until reviewed
To review: set your handle → choose ok (keep) or drop (remove) + a reason → Apply & submit to main writes it to the register, re-gates, and opens a PR that auto-merges on green CI. No live server? Use Download decisions CSV → review.py apply.
Reason codes — what each means
smuggled_number — number not in this quote
cross_row_stitch — two table rows merged into a range
wrong_variable — mis-labelled variable (SOM≠SOC)
wrong_practice — wrong NbS/practice (reforestation, pure forestry, carbon-only)
species_envelope — per-species range, not a practice rule
table_garble — mangled table extraction
off_scope — not a suitability claim (site descriptors, wrong crop)
wrong_table — soft enabling-env → T3/M2b/next-steps, not T4
constrained_aoi — source constrains NbS to marginal/residual land → understates opportunity-space extent
quote_too_narrow — too little context
relationship_missed — suitability curve/directionality in source but not captured
insufficient_context — no caption/context or screengrab failed → cannot review
uninterpretable_weight — weight/score with no scale (“5 out of what?”)
speculative_evidence — speculation/assumption, not first/second-hand
false_flag — verifier wrong → keep (ok)
accepted_correction — fix accepted → ok
confirmed_pass — AI pass was correct → ok (passed mode)
missed_error — AI missed it → drop (passed mode)
other — explain in the note
