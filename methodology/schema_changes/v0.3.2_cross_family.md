# Schema change v0.3.2 — `agroforestry__cross_family` suitability-family value

> **Status:** applied (FAM sentinel row + EV re-tag); **pending team ratification** of the
> locked 5-family scheme. Paste to a GitHub issue and tag Namita + MFL.

## Why
The locked scheme has 5 families (F1–F5), each keyed to a *shared dominant limiting factor*.
But many evidence claims are **general biophysical-envelope facts** (slope, rainfall, temp,
soil) that apply across ALL families, not one. Forcing one `suitability_family_id` per row
dumped 75% of evidence into F1 (planted_silvoarable), starving F2/F4 and skewing any
per-family synthesis.

## Change
- New value `agroforestry__cross_family` (FAM sentinel row `cross_cutting_envelope`,
  `spatial_product_type=qualitative_only`) for family-agnostic envelope evidence.
- Re-tagged 81 EV units the extractors had flagged "general — applies cross-family"
  (previously force-pinned to F1) → `agroforestry__cross_family`.
- Fixed 13 mis-formatted ids (`F1_planted_silvoarable` → `agroforestry__planted_silvoarable`).
- EXTRACT_SPEC updated: general envelope → cross_family, not the F1 default.

## Open for the team
Is `cross_family` the right model, or should a general claim be **replicated per applicable
family**? cross_family keeps one row + clean provenance; replication makes per-family counts
self-contained but duplicates evidence. Decision needed before T4 synthesis consumes families.
