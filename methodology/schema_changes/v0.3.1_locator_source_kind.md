## Why

The evidence layer could only verify PDF page numbers. To register **websites, grey
literature, tools, and codebases** with the same rigor (verbatim quote at a precise,
re-checkable anchor), the locator had to generalise beyond `page`, and each source
needs a declared kind + a pointer to its cached artifact. This is the structural
half of the post-contamination hardening (see `schema/registers/_quarantine/` and
`schema_tools/validate_sources.py`).

## Change

### EV â€” Evidence Register
- **`page`** moves from **required â†’ conditional** (required only when
  `locator_type = page`).
- **+`locator_type`** (required, enum): `page | section | char_span | file_line` â€”
  selects which anchor the verifier checks.
- **+`locator`** (conditional): the anchor for non-page types â€” section heading,
  `"start-end"` char offsets, or `"path:line[-line]"`.
- **+`commit_sha`** (conditional): **required when `locator_type = file_line`** â€”
  pins a codebase quote to an immutable commit (code drifts; a bare `path:line` is
  not provenance).

### SRC â€” Source Register
- **+`source_kind`** (required, enum): `paper | grey_lit | website | tool | codebase`.
- **+`url`** (optional): canonical resource URL when there is no DOI.
- **+`artifact_sha1`** (optional): sha1 of the cached artifact in `.cache/corpus/`.
- **+`retrieved_at`** (optional): fetch date (web sources change â€” pin the snapshot).
